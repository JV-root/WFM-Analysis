# 📊 WFM Analysis – Call Center Performance (October 2020)

## 📌 Overview

This project presents an end-to-end data analysis of call center operations during October 2020, focusing on Service Level Agreement (SLA) performance, workload distribution, and customer experience.

The goal is to transform raw operational data into actionable insights, supporting decision-making around efficiency, capacity planning, and incident management.

---

## 🎯 Objectives

This analysis was designed to answer key operational questions:

- Where do SLAs currently stand, and how far are we from the target?
- Is performance driven by isolated events or structural issues?
- How is workload distributed across call centers?
- What are the main drivers impacting customer experience?
- What actions can improve SLA and operational efficiency?

---

## 📂 Dataset

- Source: Internal Call Center Data (simulated dataset from Kaggle)
- Period: October 2020
- Volume: ~32k interactions
- Key features:
  - SLA status (Within / Below / Above)
  - Call center location
  - Sentiment
  - CSAT score
  - Call duration (AHT)
  - Contact reason (including service outages)

---

## 🧠 Analytical Approach

The analysis follows a structured storytelling framework:

1. **Performance Diagnosis (SLA)**
2. **Root Cause Analysis (Workload Distribution)**
3. **Operational & Experience Drivers**
4. **Recommendations**

This approach ensures that insights evolve from *symptoms → causes → actions*.

## 📈 Visualization Standards

This project applies advanced data visualization best practices:

- Time series normalization (complete date range)
- Percentage-based distributions for comparability
- Highlight vs. secondary series styling
- Direct labeling (no legends)
- Minimalist design (no top/right borders)
- Executive-style titles and subtitles
- Explicit source and period annotation
